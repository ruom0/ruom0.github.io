虚拟机环境配置

虚拟机系统为ubuntu 18.04.5，只安装了Python3

pwntools安装

  1. 执行命令 sudo apt install python3-pip 安装 pip
  2. 执行命令 pip install pwntools 安装 pwntools

pwndbg及peda安装

从pwndbg以及peda下载两个工具的压缩包，将它们都解压在用户目录下。

cd pwndbg进入pwndbg的目录，执行命令./setup.sh，脚本运行完成后执行gdb。（这边遇到一个不知道为什么的问题，gdb进入后没有变pwndbg，查看了一下.gdbinit文件发现pwndbg的gdbinit.py文件没有写进去，自己手动加了一行source ~/pwndbg/gdbinit.py）

执行命令"source ~/peda/peda.py" >> ~/.gdbinit将peda的启动方式写进去。

更换pwndbg和peda的方法为将另一个的source给注释掉，如果两个同时存在则只会启动第一行所指定的插件。

LibcSearcher安装

从LibcSearcher下载压缩包解压，进入目录中执行sudo python setup.py develop即可。

Winafl安装

  1. Visual Studio 2017安装

    由于官网比较难下载到，因此在其他网站下载了2017的安装包并安装。

  2. CMake安装

    从CMake官网下载CMake的msi文件安装

  3. DynamoRIO下载

    1. 前往DynamoRIO官网下载压缩包
    2. 解压压缩包

  4. 打开VS 2017的 x64_x86 交叉工具命令提示符，进入winafl的build64目录中

  5. 输入命令cmake -G"Visual Studio 15 2017" -A x64 .. -DDynamoRIO_DIR=E:\DynamoRIO\cmake -DINTELPT=0 -DUSE_DRSYMS=1，输出结果如下：

  6. 打开build64目录下的WinAFL.sln，单击菜单""生成"中的生成解决方案

  7. 右键右侧搜索解决方案资源管理器中的afl-fuzz，选择设为启动项目

  8. 右键afl-fuzz，选择属性，将调试中的命令参数属性改为-i input -o output -D E:\DynamoRIO\bin64 -t 20000 -- -coverage_module test.exe -fuzz_iterations 5000 -target_module test.exe -target_method main -nargs 2 -- test.exe @@

  9. 修改工作目录为$(ProjectDir)bin\Debug

  10. 在"调试属性"–>“C/C++”–>"常规"中将调试信息格式属性改为用于"编辑并继续"的程序数据库(/ZI)

  11. 在"调试属性"–>“C/C++”–>"优化"中将优化属性改为已禁用(/Od)

  12. 在"调试属性"–>“链接器”–>"调试"中将生成调试信息属性改为生成调试信息(/DEBUG)

  13. 重新生成解决方案
